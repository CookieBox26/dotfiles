最新の打合せ資料をベースに、次の打合せ資料を作成してください。
以下の手順にしたがってください。

- 最新の打合せ資料を特定してください。
  `backyard/Mtg/YYYYMMDD/slide.tex` のうち最新日付のものです。
- 次回の打合せ日付を特定してください。
  最新の打合せ資料の「次回打合せ:」の箇所に記入してあります。
  もし特定できなければユーザに確認してください。
- 最新の打合せ資料をベースに、次回の打合せの日付をサブディレクトリ名にして
  `backyard/Mtg/YYYYMMDD/slide.tex` を作成してください(Bash ツールによる
  ディレクトリ作成は許可していないので Write ツールで直接作成してください)。
  このとき、最新の打合せ資料から以下を修正してください。
  見出しは `\myheader{}` のように記入されています。
  - 「進捗報告 (YYYY/MM/DD)」セクション:
    - セクション名の日付を次の打合せの日付にしてください。
    - 見出し「前回報告 (YYYY/MM/DD) のトピック」について、
      見出し名の日付を最後の打合せの日付に更新し、
      内容を「今回報告 (YYYY/MM/DD) のトピック」から転記してください。
      ただし、「次回打合せ:」というアイテムは省いてください。
    - 見出し「要対応事項」は、据置きにしてください。
    - 見出し「わかったこと」は、何か内容があれば
      「これまでにわかったこと＆気になったこと」セクションの
      見出し「これまでにわかったこと」の末尾に転記してください。
      その後、箇条書きのアイテム 1 つ目を「ここに内容を書く」として残し、
      それ以降は削除してください。
    - 見出し「今回報告 (YYYY/MM/DD) のトピック」について、
      見出し名の日付を次回の打合せの日付に更新し、
      箇条書きアイテム 1 つ目の「次回打合せ:」の日付部分を削除してください。
      アイテム 2 つ目は内容を「ここに内容を書く」に書き換えて残し、
      それ以降は削除してください。
    - 見出し「会議」は据置きにしてください。
      ただし、「開催期間」「申込期間」の列について、
      「期間終了日 < 次回打合せ日付」になっているものは、
      `\grey` を付けてください (既に付けてあるものに倣ってください)。
  - 「マイルストーン」セクション:
    - 据置きにしてください。
  - 「これまでにわかったこと＆気になったこと」セクション:
    - 据置きにしてください。
  - それ以降のセクション:
    - セクションごと削除してください。
- 最後に、ビルド設定スクリプト `backyard/config.sh` の、
  関数 `get_config()` 内の変数 `yyyymmdd` の値を
  次回の打合せ日付に更新してください。
